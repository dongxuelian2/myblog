# Design System — Cold Signal

## 设计哲学

五个关键词：**冷峻、精确、克制、连贯、锐利**。

视觉语言参考东京地铁标识系统——用几何引导视线，用功能色标识状态，用留白创造秩序。页面不是画廊，而是一套信息导引系统：每个元素都有存在的理由，没有理由的就不该出现。

核心原则：

- **单色主体**：页面 95% 的色彩是黑、白、灰三档。彩色只在需要"信号"的位置出现。
- **结构一致性**：所有页面共享同一套骨架——Header、内容区、Footer。页面之间的切换是"原子变化"：只有必须变的组件才变，其余保持不动。
- **几何节奏**：间距、比例、对齐都基于固定的网格。没有随意的魔法数字。
- **排版即设计**：字体层级、字重对比、行距控制本身就是视觉表达，不依赖装饰性元素。
- **功能色，非装饰色**：信号红 `#c53030` 只用于标记状态（active、hover、indicator），不填充大面积背景。

## 品牌身份

- 站点名称：**夏**（单字，冷峻，不加缀词）
- 作者：夏
- 头像：`/images/avatar.svg`——几何抽象图形，不是照片
- 语调：冷静、直接、不寒暄

## 设计令牌

### 色彩

纯冷色系。没有暖调、没有米色、没有柔和渐变。

| Token            | 值        | 用途                                 |
| ---------------- | --------- | ------------------------------------ |
| `paper`          | `#fafafa` | 全站背景，纯冷白                     |
| `surface`        | `#ffffff` | 卡片、输入框内部                     |
| `surface-muted`  | `#f2f2f2` | 次级分组、代码块背景、占位区         |
| `ink`            | `#0c0c0c` | 主标题、品牌、关键文案               |
| `ink-soft`       | `#3d3d3d` | 正文                                 |
| `ink-subtle`     | `#8b8b8b` | 辅助信息、时间、标签                 |
| `outline`        | `#e5e5e5` | 边框、分割线                         |
| `outline-strong` | `#c0c0c0` | hover / active 边框                  |
| `accent`         | `#c53030` | 信号红——唯一彩色，极少量使用         |
| `accent-soft`    | `#fef2f2` | 信号红浅底，仅用于 focus-ring / 选中 |

约束：

- `accent` 只允许出现在：页面顶部 2px 指示线、链接 hover、active 导航下划线、blockquote 左边线、focus ring。
- 按钮、卡片、标签、背景不允许使用 accent 做大面积填充。
- 不使用渐变、不使用 frosted glass（header backdrop-blur 除外）。

### 字体

| 角色     | 字体族                   | 用途                           |
| -------- | ------------------------ | ------------------------------ |
| `sans`   | Noto Sans SC 系          | UI 标题、正文、导航、按钮      |
| `serif`  | Noto Serif SC 系         | 仅文章正文内 h2/h3             |
| `mono`   | JetBrains Mono           | 日期、标签、代码、元数据       |
| `reading`| Noto Sans SC 系          | 长文正文                       |

约束：

- 字重只用 400 / 500 / 600 三档，不使用 700+。
- 元数据（日期、阅读时间、标签）统一使用 mono 字体，传达技术精确感。
- Serif 仅在 `.article-prose` 内的 h2/h3 出现。

### 宽度

| Token     | 值       | 用途                     |
| --------- | -------- | ------------------------ |
| `shell`   | `80rem`  | 全站最外层容器           |
| `layout`  | `68rem`  | 页面内容主区域           |
| `reading` | `44rem`  | 正文阅读列               |

### 间距节奏

固定 6 档，不出现游离值：

- `4px` — 极细内部间距
- `8px` — 控件内部
- `16px` — 组件内部
- `24px` — 区块内部
- `48px` — 模块间距
- `80px` — 页面大段落

### 圆角与阴影

- 卡片：`rounded-lg`（8px）
- 按钮 / 输入框：`rounded-md`（6px）
- 代码块：`rounded`（4px）
- 阴影：几乎不用。仅 `shadow-sm` 在卡片 hover 时出现。

## 页面骨架

### 全局结构

```
┌─────────────────────────────────────────────┐
│ ▇▇ 2px accent 指示线                        │
├─────────────────────────────────────────────┤
│ Header: 品牌 ─────────────────── 导航       │
├─────────────────────────────────────────────┤
│                                             │
│              页面内容区                       │
│                                             │
├─────────────────────────────────────────────┤
│ Footer: 品牌 · 简介 ─────── 版权 · 技术栈   │
└─────────────────────────────────────────────┘
```

所有页面共享这套骨架。Header 和 Footer 完全不变。页面切换时，只有中间内容区发生变化。

### Header

- 高度 56px
- 左：品牌单字"夏"，`text-lg font-semibold tracking-tight`
- 右：导航项（首页 / 文章 / 关于），mono 字体，小号
- Active 导航项：accent 色下划线
- 半透明背景 + backdrop-blur

### Footer

- 左列：品牌"夏" + 一句话简介
- 右列：版权年份 + 技术栈
- 极简，两行以内

### 首页

结构：
1. Hero 区：大标题 + 一句话描述（无"原则卡片"废话）
2. 精选文章：单篇大卡片
3. 最近文章列表：紧凑的行式列表（标题 + 日期 + 标签），不是网格卡片
4. 底部入口：到文章归档的链接

不允许：
- 堆砌"设计原则"等自说自话的内容
- 同时出现多种不同样式的卡片
- 大面积空白区域没有内容支撑

### 文章归档页

结构：
1. 页面标题行：标题 + 搜索
2. 侧栏：排序 + 分类标签（垂直堆叠，mono 字体，中文标签）
3. 主区域：统一列表

不允许：
- "Filter and sort" 等英文标签（全部中文化）
- 独立的大 Hero 区（标题即够）
- 精选卡片重复出现在归档页

### 文章详情页

结构：
1. 返回链接
2. 标题区：分类标签（mono） + 标题 + 摘要
3. 元信息行：日期 + 阅读时间 + 标签（一行排列，mono 字体）
4. 封面图（如有）
5. 正文
6. 文章底部：上一篇 / 下一篇导航

不允许：
- 右侧独立的元信息栏（信息密度太低，浪费空间）
- Category 区域的异常间距
- "Share" / "Reading time" 等英文标签

### 关于页

结构：
1. 头像 + 名称 + 一句话介绍
2. 正文内容（Markdown 风格的自我介绍）
3. 联系方式 / 链接

不允许：
- 解释"这个博客的设计系统"——关于页是关于作者，不是关于页面本身

## 组件规范

### 顶部指示线

- 固定在页面最顶端
- `height: 2px; background: accent`
- 全宽，不受容器限制
- 这是整个站点唯一的大面积彩色元素

### 文章卡片

两种变体共享同一基础结构：

**精选卡片（feature）**
- 水平布局：左图（4:3）+ 右内容
- 标题 `text-2xl font-semibold`
- 有摘要

**列表卡片（list）**
- 一行布局：标题 + 日期 + 标签
- 标题 `text-base font-medium`
- 无摘要，无图片
- 干净的行式排列，靠分割线区隔

共同规则：
- 日期和标签使用 mono 字体
- hover 时标题变 accent 色
- 无阴影，无浮起效果
- 无图时使用 `surface-muted` 背景 + 首字缩写

### 标签

- mono 字体
- 带 `#` 前缀
- 小号（`text-xs`）
- 不使用彩色背景
- 行内显示，间距紧凑

### 按钮

仅三类：
- **Primary**：`bg-ink text-paper rounded-md`，极少使用
- **Secondary**：`border border-outline rounded-md`
- **Text Link**：下划线链接，hover 时 accent 色

### 搜索框

- `rounded-md border border-outline`
- mono 字体的 placeholder
- focus 时 `border-outline-strong` + accent focus ring

## 正文排版

- 字号：`17px`
- 行高：`1.9`
- 列宽：`44rem`
- 段距：`1.5em`
- 文章标题：serif
- 正文：sans (reading font)
- 引用：accent 左边线 2px + `surface-muted` 背景
- 行内代码：`surface-muted` 底色 + `outline` 边框
- 代码块：`surface-muted` 底色 + 可横滚

## 交互

- 动效时长：`150ms - 200ms`
- 缓动：`cubic-bezier(0.22, 1, 0.36, 1)`
- 只允许颜色、边框、下划线变化
- 不允许缩放、位移、弹跳

## 页面切换一致性规则

1. Header / Footer 在所有页面完全一致，0 变化。
2. 内容区的容器宽度保持一致（`max-w-layout`）。
3. 不同页面之间的标题层级对应关系一致：页面名 = `h1`，区块名 = `h2`。
4. 文章详情页的标题区与归档页的标题区在字号、间距上保持对应。
5. 导航的 active 状态能在所有页面正确反映当前位置。

## 新增内容 / 组件检查清单

1. 是否使用了现有的设计令牌？
2. 是否保持了结构一致性？
3. 彩色是否只出现在允许的位置？
4. 元数据是否使用了 mono 字体？
5. 是否避免了新的"废话段落"或占位内容？
6. 中文标签是否完整（没有遗漏的英文 UI 文案）？
